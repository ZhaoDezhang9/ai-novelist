# AI 小说家

> 一句话灵感，换一本完整小说。

扔进来一个想法——"物理学家穿越到修仙世界用科学重新定义修炼体系"——坐下来喝杯茶，AI 小说家会帮你写完大纲、设定世界观、塑造角色、一章接一章地生成文本，自动质检、自动改写、自动管理伏笔，直到整本书完稿。

## 它能干什么

### 你说句话，它写一本书

不会只扔给你一段 AI 文字就完事。整个流程是：

1. **理解你的灵感** → 自动生成标题、主题、三幕式大纲
2. **构建世界** → 世界观圣经、角色关系网、伏笔清单
3. **逐章创作** → 每章按大纲推进，角色性格不崩
4. **自动质检** → 写完自己检查：前后矛盾了吗？抄模板了吗？情绪节奏对吗？
5. **定向改写** → 只改有问题的地方，不是整章重来
6. **记忆管理** → 记住前面写了什么，不会写到最后忘了主角是谁

### 书房不只是看字

打开书房页，除了阅读文本，还能：

- **质量雷达图** — 一致性、原创性、大纲对齐度，一目了然
- **情绪曲线** — 紧张/放松/悲伤/愉悦四维度追踪整本书的情绪节奏
- **角色关系图谱** — 谁和谁是道侣，谁在暗算谁，一张图看清楚
- **大纲视图** — 三幕结构，每章对应什么节拍，哪里写完哪里没写完

### 写作风格不是玄学

类型、文风、视角都能量化控制。想写"古典雅致的修仙文"还是"冷峻写实的科幻风"，选一下就行。

## 本地跑起来

```bash
# 1. 安装依赖
pip install -r requirements.txt
cd frontend && npm install && cd ..

# 2. 配置 API Key
cp .env.example .env
# 编辑 .env，把 NOVELIST_LLM_API_KEY 换成你的 Key

# 3. 启动
./启动.bat
```

浏览器打开 **http://localhost:3000**

## 项目结构

```
ai-novelist/
├── backend/               # FastAPI 后端
│   ├── api/               # REST 接口
│   ├── core/              # 编排器、LLM 客户端
│   ├── generation/        # 大纲引擎、章节创作
│   ├── memory/            # 分层记忆、上下文管理
│   ├── quality/           # 质检流水线
│   ├── management/        # 伏笔、改写、风格
│   └── pipeline/          # 并行调度
├── frontend/              # React 前端
│   └── src/
│       ├── pages/         # 小说集、创作、书房、设定
│       ├── components/    # 通用组件
│       └── styles.ts      # 设计系统
└── .env.example           # 配置模板
```
